Module core.bheap

A light implementation of binary heaps.

While running a search, A* algorithm maintains a list of nodes called open list. Finding in this list the lowest cost node from the node being processed can be quite slow, (as it requires to skim through the collection of nodes stored in this list) especially when dozens of nodes are being processed (large maps).

The current modules implements the binary heap data structure, from which the internal open list will be instantiated. As such, looking up for lower-cost node will run faster, and globally makes the search algorithm run faster.

This module should not normally be used explicitely. The algorithm uses it internally.
