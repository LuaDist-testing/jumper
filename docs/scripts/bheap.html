Script bheap

A light implementation of binary heaps.

The graph search algorithm implemented here is a mix of Jump point Search and A* algorithms. A* works by maintaining a list of nodes called open list. Finding in this list the lowest cost node from the node being processed is a slow operation, as it requires to skim through the collection of nodes stored in this list.

The current modules implements a Binary Heaps class, a priority queue data structure. As such, lookup for lowest-cost node is faster, and globally makes the search algorithm run faster.

This module should not normally be used explicitely. The algorithm uses it internally.
